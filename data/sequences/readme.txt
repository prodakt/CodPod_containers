This directory contains full coding (cds) and non‑coding (nc) sequence datasets for three vertebrate species: Homo sapiens, Mus musculus and Sus scrofa. Coding and non‑coding sequences for each species were split into training and test subsets in a 60/40 ratio (60% training, 40% test).

Sequences are organized under:

data/sequences/Homo_sapiens/

data/sequences/Mus_musculus/

data/sequences/Sus_scrofa/

Each species directory contains several FASTA files following a common naming scheme.

Source data
The reference transcript sequences were downloaded from Ensembl release 115 (GENCODE/Ensembl annotation for Homo sapiens, Mus musculus and Sus scrofa). Coding and non‑coding transcripts were extracted and exported as species‑specific FASTA files:

*_cds.all.fa — all coding sequences (CDS) for the given species.

*_ncrna.fa — all non‑coding RNA sequences (ncRNA) for the given species.

*_cds.all_pc_labeled.fa — coding sequences labeled as protein‑coding (pc).

*_ncrna_nc_labeled.fa — non‑coding sequences labeled as non‑coding (nc).

These labeled source files form the basis for all downstream train/test splits and derived subsets.

Train and test splits
For each species, the labeled coding and non‑coding sequences were split into training and test sets using a 60/40 ratio (60% of sequences assigned to training, 40% to test):

pc__train.fasta — protein‑coding training set.

pc__test.fasta — protein‑coding test set.

nc__train.fasta — non‑coding training set.

nc__test.fasta — non‑coding test set.

Train/test splits are stratified by class (pc/nc) and respect the original labeling derived from Ensembl release 115.

Balanced training sets
To enable training under class‑balanced conditions, additional balanced training sets were constructed by downsampling the majority class to the size of the minority class. As a result:

nc__train_balanced.fasta — non‑coding training set balanced to match the number of coding sequences in the corresponding training set.

pc__train_balanced.fasta — coding training set downsampled so that the number of pc sequences matches the number of nc sequences.

In each species directory, pc__train_balanced.fasta and nc__train_balanced.fasta contain the same number of sequences and can be used to train models under balanced class conditions.

Test datasets for prediction
Test data are kept in their original, imbalanced form in order to better reflect realistic prediction scenarios:

seqs2predict__full.fasta — full test set for prediction. It contains all sequences from the test subset (both coding and non‑coding), with the original class proportions preserved. No balancing is applied here.

seqs2predict__small.fasta — compact, balanced validation set for quick evaluation. It consists of 500 coding (pc) and 500 non‑coding (nc) sequences sampled from the test data. This file is intended for rapid sanity‑checks, method prototyping and lightweight benchmarking without loading the full test set.

Thus, seqs2predict__full.fasta supports evaluation in a natural, potentially skewed class distribution, whereas seqs2predict__small.fasta provides a convenient balanced subset for fast comparison of methods.

Reduced training subsets (train_small)
In addition to the full training sets, reduced training subsets are provided for each species:

pc__train_small.fasta — reduced protein‑coding training subset.

nc__train_small.fasta — reduced non‑coding training subset.

These files were generated by random sampling without replacement from the corresponding full training sets (pc__train.fasta, nc__train.fasta). The typical generation procedure is:

bash
python3 make_small_train.py <pc_train.fasta> <nc_train.fasta> <seed> <n_seqs>
For the datasets in this repository:

input files: pc__train.fasta and nc__train.fasta (species‑specific),

output files: pc__train_small.fasta and nc__train_small.fasta,

sampling mode: random sampling without replacement,

random seeds: 42 for the coding (pc) subset and 43 for the non‑coding (nc) subset within a given run,

target size: 1000 sequences per output file (i.e. the last argument n_seqs specifies number of sequences, not total nucleotide length).

Because sampling is reproducible with a fixed seed and given input files, each *_train_small.fasta can be regenerated identically if needed.

Summary of use cases
Full train/test (pc__train.fasta, nc__train.fasta, pc__test.fasta, nc__test.fasta): standard training and evaluation.

Balanced train (pc__train_balanced.fasta, nc__train_balanced.fasta): training under balanced class conditions.

Full test (seqs2predict__full.fasta): realistic evaluation with original pc/nc proportions.

Small test (seqs2predict__small.fasta): fast, balanced validation (500 pc + 500 nc).

Reduced train (pc__train_small.fasta, nc__train_small.fasta): quick experiments, debugging and prototyping with smaller but representative subsets.

Using the datasets with Git LFS
Full FASTA files are stored via Git Large File Storage (LFS) due to their size. To clone the repository and download all data:

bash
git clone https://github.com/prodakt/CodPod_containers.git
cd CodPod_containers
git lfs install
git lfs pull
After these steps, all FASTA files in data/sequences/ will be available locally in their full size and can be directly used as input for classification tools and pipelines.
